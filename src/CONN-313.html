As a CONNECT adopter, I want CONNECT 4.0 to be tested on WebSphere 7.x using JDK 1.6 to validate that CONNECT will work in my preferred enviornment

Description

HISTORY:
CONNECT 4.0 supports multiple application servers. For details on the app servers and OS environments that the 4.0 release was tested on refer to the release testing wiki link- https://developer.connectopensource.org/display/CONNECT40/Release+Testing. The WS environments supported by this release include WebSphere Enterprise 8.5 and WebSphere CE 3.0.0.2. CMS is using WAS 7 and JDK 1.6 for the OFM program. SSA plans to WAS 7 as well and will plan on taking 4.0 in the next 6-18 months. CMS is concerned about using CONNECT on a version that is not supported and requires that we test and support using their environment(WAS 7/JDK 1.6). We received some input from CMS that JDK 1.7 may not work with WAS 7 or is not supported for WAS 7. THis was tested as part of CONN-279 and it was validated that it is indeed the case. On the other side JDK 1.6 is no longer supported and/or public updates may not be available after Feb 2013. However CMS does not see that them moving to WAS 8.5 for the next 5 years. While they would like to use 4.0 in the immediate future, they are concerned that they do not want to use it in an environment that is not supported or has not been tested. SSA has told us that WAS bundles 7.x with JDK 1.6 and hence they would need support in that environment as well.

The goal is to support our partners in their preferred environment, else they will not be able to use the features possible with 4.0.

CONSIDERATION/DETAIL:
THIS IS A RESEARCH TICKET TO UNDERSTAND THE IMPACT OF MODIFYING 4.0 CODE TO SUPPORT JDK 1.6.
CONNECT 4.0 code base uses JDK 1.7. Test 4.0 on WAS 7 using JDK 1.6 and assess the LOE to make this change.
We need to also know if there any features that will not work if JDK 1.6 is used.

CONSTRAINTS:
1)The development team would need to acquire licenses for appropriate versions of the WAS application server that the CMS OFM team uses.
2)Work with the CMS dev team if there are any questions

BUSINESS VALUE:
Support partners as they move to CONNECT 4.0 in their preferred environment(WAS 7/JDK 1.6).
Help estimate level of effort to add changes to support this environment.

COMPLETION CRITERIA:
Test WAS 7 running using JDK 1.6. Document how this was done and issues faced.
Note particularly if any features we think will not work as desired.
Estimate Level of Effort to make this change in the 4.0 code base.
